Phase 02: Monitoring & Observability

DOCUMENT CATEGORY: Runbook SCOPE: Monitoring and observability PURPOSE: Configure Azure Monitor, Log Analytics, HCI Insights, and Update Manager MASTER REFERENCE: Microsoft Learn - Monitor Azure Local

Status: Draft

This stage configures comprehensive monitoring and observability for Azure Local clusters and Arc-enabled servers. These configurations are applied post-deployment when Arc-enabled resources exist.

Steps Overview​

StepTitleDescriptionDependencies
1Configure Log Analytics WorkspaceCreate and configure the Log Analytics workspace for all monitoring dataNone
2Configure Azure Monitor AgentDeploy AMA to cluster nodes, configure Data Collection RulesStep 1
3Enable HCI InsightsEnable Insights workbook for cluster health monitoringSteps 1-2
4Setup AlertingConfigure alert rules and action groupsSteps 1-2
5Deploy OMIMSWAC MonitoringDell hardware monitoring via Windows Admin CenterSteps 1-2
6Configure Network Device LoggingSyslog forwarding from switches and firewallsStep 1
7Configure Datadog IntegrationWDAC policies, agent deployment, dashboards, and alertingSteps 1-4
Primary Alerting Platform

Datadog is the primary source for Insights, Metrics, and Alerting for Azure Local Cloud Azure Local deployments. While Azure Monitor provides native telemetry, Datadog serves as the centralized platform for operational dashboards, alerting, and ServiceNow integration. See Task 7 for complete configuration.

Note: Azure Update Manager is covered in Phase 20: Security & Governance.

Prerequisites​

  • ✅ Cluster Deployed — Azure Local cluster registered with Azure Arc
  • ✅ Arc-Enabled Servers — Cluster nodes registered as Arc-enabled servers
  • ✅ Log Analytics Workspace — Created in Stage 06 (or create new in Step 1)
  • ✅ NDM Server — Deployed for network device logging (Step 6)
  • ✅ Windows Admin Center — Installed for OMIMSWAC (Step 5)
