Infrastructure-as-Code templates, modules, and CI/CD pipeline configurations for automated Azure Local deployments. Three deployment paths are supported, each consuming the same `variables.yml` single source of truth.

## Deployment Paths

| Path | Azure Resources | On-Prem / Guest Config | Best For |
|------|:-:|:-:|---|
| **Terraform + PowerShell** | Terraform | PowerShell scripts | Teams with existing PowerShell expertise |
| **Terraform + Ansible** | Terraform | Ansible playbooks | Teams wanting cross-platform config management |
| **Ansible Only** | `azure.azcollection` | Ansible playbooks | Teams standardizing on Ansible |

### Why Three Paths?

Terraform manages Azure resources (management groups, VNets, Key Vault, VMs, monitoring, policy), but **cannot** manage:

- On-premises Active Directory (OU creation, security groups, DNS records)
- Bare-metal OS configuration (hostname, NIC naming, NTP, domain join)
- Azure Arc agent bootstrapping on physical nodes
- Guest OS monitoring agent installation
- Network switch and BIOS/iDRAC configuration

PowerShell or Ansible fills this gap. Choose the path that matches your team's skills.

## Quick Start

### Path 1: Terraform + PowerShell

```bash
# 1. Bootstrap remote state
cd src/terraform/backend && terraform init && terraform apply

# 2. Deploy Azure foundation
cd src/terraform/environments/azure-local
terraform init && terraform plan && terraform apply

# 3. Run post-Terraform PowerShell configuration
pwsh scripts/deploy/common/Invoke-PostTerraformConfiguration.ps1 \
  -ConfigPath config/variables/variables.yml
```

### Path 2: Terraform + Ansible

```bash
# 1-2. Same as Path 1 (Terraform for Azure resources)

# 3. Run Ansible for on-prem configuration
cd src/ansible
ansible-galaxy collection install -r collections/requirements.yml
ansible-playbook -i inventory/hosts.yml playbooks/site.yml
```

### Path 3: Ansible Only

```bash
cd src/ansible
ansible-galaxy collection install -r collections/requirements.yml
ansible-playbook -i inventory/hosts.yml playbooks/site.yml
```

## Variable System

All automation consumes the central `config/variables/variables.yml` file. Export functions transform YAML into tool-specific formats:

```powershell
. scripts/common/utilities/helpers/config-loader.ps1
$config = Get-Config -ConfigPath "config/variables/variables.yml"

Export-TerraformTfvars -Config $config -OutputPath "src/terraform/environments/azure-local/terraform.tfvars"
Export-AnsibleVars -Config $config -OutputPath "src/ansible/inventory/group_vars/all.yml"
```

## Repository

All automation code lives in the [azurelocal-toolkit](https://github.com/AzureLocal/azurelocal-toolkit) repository:

| Directory | Contents |

| `src/terraform/` | Terraform modules and root environment |
| `src/ansible/` | Ansible roles, playbooks, and inventory |
| `pipelines/` | CI/CD pipeline definitions (GitLab, GitHub, Azure DevOps) |
| `scripts/deploy/` | PowerShell deployment scripts |
| `config/variables/` | Central variable system |
