This stage configures licensing options and telemetry settings to optimize costs and enable comprehensive monitoring for Azure Local clusters.

## Overview

Proper licensing and telemetry configuration ensures:
- **Cost Optimization**: Azure Hybrid Benefit reduces compute costs by leveraging existing licenses
- **License Compliance**: Windows Server Subscription provides proper licensing for VMs
- **Operational Visibility**: Enhanced telemetry enables proactive support and diagnostics
- **EU Compliance**: Proper data residency configuration for European deployments

## Steps in This Stage

| Step | Title | Description |
|------|-------|-------------|
| 1 | [Enable Azure Hybrid Benefit](task-01-enable-azure-hybrid-benefit.mdx) | Configure Azure Hybrid Benefit to reduce VM compute costs |
| 2 | [Activate Windows Server Subscription](task-02-activate-windows-server-subscription.mdx) | Enable Windows Server subscription for VM licensing |
| 3 | [Configure Enhanced Telemetry](task-03-configure-enhanced-telemetry.mdx) | Set diagnostic level, streaming data, and EU location settings |

## Prerequisites

Before starting this stage, ensure:

- [ ] Azure Local cluster is registered with Azure Arc
- [ ] You have Owner or Contributor access to the Azure subscription
- [ ] Cluster is operational and passing health checks
- [ ] You have appropriate Windows Server licenses (for Azure Hybrid Benefit)

## Cost Impact

| Feature | Impact |
|---------|--------|
| Azure Hybrid Benefit | Up to 40% savings on Windows VM compute costs |
| Windows Server Subscription | Per-core licensing for unlimited VMs |
| Enhanced Telemetry | No additional cost, enables proactive support |
