## Overview

This step covers the knowledge transfer (KT) session structure and delivery. The goal is to ensure customer personnel have the knowledge and skills to operate their Azure Local environment.

---

## Pre-Session Preparation

### Scheduling

- [ ] Identify customer attendees and roles
- [ ] Schedule session(s) with adequate time
- [ ] Send calendar invites with agenda
- [ ] Confirm remote/on-site logistics
- [ ] Verify customer has Azure Portal access

### Materials Preparation

- [ ] Training slide deck prepared
- [ ] Handover documentation package ready
- [ ] Lab environment available (if applicable)
- [ ] Screen sharing / recording setup confirmed
- [ ] Quick reference guides printed/distributed

---

## Knowledge Transfer Agenda

### Session 1: Environment Overview (1 hour)

| Topic | Duration | Content |
|-------|----------|----------|
| Solution Architecture | 20 min | Physical layout, network design, Azure integration |
| Azure Portal Navigation | 20 min | Accessing cluster, viewing health, finding resources |
| Documentation Walkthrough | 20 min | Review of delivered documentation package |

### Session 2: Day-to-Day Operations (2 hours)

For detailed training curriculum, see [Client Training and Handover](../../08-lifecycle-operations/client-training-handover.mdx).

| Topic | Duration | Content |
|-------|----------|----------|
| VM Deployment | 30 min | Creating VMs from Azure Portal |
| VM Management | 30 min | Start/stop, resize, disk management |
| Backup Operations | 30 min | Backup policies, on-demand backup, restore |
| Site Recovery | 30 min | Replication, test failover, failover procedures |

### Session 3: Monitoring & Maintenance (1 hour)

| Topic | Duration | Content |
|-------|----------|----------|
| Health Monitoring | 20 min | Viewing cluster health, alerts, insights |
| Update Management | 20 min | Azure Update Manager, update scheduling |
| Support Procedures | 20 min | Azure Local Cloud support contacts, escalation paths |

---

## Hands-On Exercises

Include practical exercises where customers perform tasks:

### Exercise 1: Deploy a Test VM
- Customer creates a new VM from Azure Portal
- Configure networking and storage
- Verify VM is running and accessible

### Exercise 2: Configure Backup
- Customer assigns backup policy to test VM
- Trigger on-demand backup
- Verify backup job completion

### Exercise 3: Enable Replication
- Customer enables ASR for test VM
- Monitor initial replication
- Perform test failover

---

## Session Delivery Best Practices

### For Remote Sessions

- Enable recording for customer reference
- Use screen sharing for demonstrations
- Pause regularly for questions
- Provide breakout time for hands-on exercises
- Share recording and materials after session

### For On-Site Sessions

- Ensure adequate seating and displays
- Provide printed quick reference guides
- Allow time for physical environment walkthrough
- Include iDRAC/hardware console demonstrations
