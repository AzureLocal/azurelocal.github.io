# Task 01: Networking & Azure Cloud Confirmation

[![Runbook](https://img.shields.io/badge/Type-Runbook-blue?style=flat-square)](./index.mdx)
[![Azure](https://img.shields.io/badge/Platform-Azure_Local-0078D4?style=flat-square&logo=microsoftazure)](https://learn.microsoft.com/en-us/azure/azure-local/)

> **DOCUMENT CATEGORY**: Runbook 
> **SCOPE**: Delivery confirmation 
> **PURPOSE**: Confirm networking and Azure components are ready for handover 
> **MASTER REFERENCE**: [Azure Local Cloud - Handover Standards](https://Azure Local Cloud.com)

---

## Overview

This step confirms that the **Infrastructure** and **Operations** teams have completed their respective networking and Azure cloud component configurations and are ready for customer handover.

:::note
Technical validation procedures are performed in [Part 7: Cluster Testing & Validation](../../06-testing-validation/). This step confirms those validations are complete.
:::

---

## Networking/Firewall Confirmation

**Delivery Team**: Infrastructure

**Process Note**: Existing network and security turnover processes apply. Nothing net new for Azure Local specifically for customer turnover from a network or security perspective.

### Confirmation Checklist

| Component | Confirmed | Notes |
|-----------|-----------|-------|
| Firewall rules configured for Azure Local traffic | ☐ | |
| DHCP scopes configured for management network | ☐ | |
| DHCP scopes configured for customer networks | ☐ | |
| TOR switches configured for RDMA/RoCEv2 | ☐ | |
| Azure VPN Gateway connected | ☐ | |
| BGP addresses visible and routes propagating | ☐ | |

---

## Azure Cloud Components Confirmation

**Delivery Team**: Operations

**Process Note**: Current workflow patterns will be utilized for customer onboarding that showcase access and environment ecosystem.

### Confirmation Checklist

| Component | Confirmed | Notes |
|-----------|-----------|-------|
| Azure subscriptions created | ☐ | |
| Azure VPN Gateway configured and connected | ☐ | |
| Azure Bastion services configured and working with Key Vault | ☐ | |
| Virtual networks (vNets) deployed and configured correctly | ☐ | |
| Management infrastructure deployed and operational | ☐ | |
| Resource groups for Azure Local Arc resources created | ☐ | |

## Troubleshooting

| Issue | Cause | Resolution |
|-------|-------|------------|
| Network connectivity validation fails | Firewall rules or NSGs blocking required traffic | Review and update network security rules per requirements |
| Azure connectivity intermittent | DNS resolution or routing issues | Verify DNS settings and network route tables |
