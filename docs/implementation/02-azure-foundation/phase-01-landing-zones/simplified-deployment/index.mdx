The **Simplified Deployment** model streamlines Azure governance by using a single management group, a single subscription, and a single resource group for all cluster resources. This is the fastest path to a working Azure Local deployment.

:::tip When to Use This Model
Choose Simplified Deployment for:
- Single Azure Local cluster deployments
- Proof of concept (PoC) and lab environments
- Environments with limited scale requirements
- Rapid deployment timelines
:::

## Architecture Overview

The example below uses Infinite azurelocal Corp (IIC). Replace names with values from your `infrastructure.yml`.

```
Tenant Root Group
└── cmp-iic-root # azure.management_groups.tenant_root.name
 └── cmp-landing-zones-iic # azure.management_groups.landing_zone.name
 └── iic-lz-azurelocal-001 # azure.subscriptions.<env>.name
 └── rg-c01-azl-eus-01 # azure_resources.resource_group_name
```

All cluster resources — Arc registration, Key Vault, storage accounts, cloud witness, and the cluster itself — reside in that single resource group. No multi-RG split is needed for the simplified model.

## Key Characteristics

| Aspect | Implementation |
|--------|----------------|
| **Management Groups** | Root MG + Landing Zone MG (from `infrastructure.yml`) |
| **Subscriptions** | Single subscription for all resources |
| **Resource Groups** | Single RG for all cluster resources |
| **Policy Scope** | Subscription-level policies |
| **RBAC Scope** | Resource group-level access control |
| **Cost Tracking** | Tag-based cost allocation |

## Comparison with Full Deployment

| Feature | Simplified | Full CAF/WAF |
|---------|------------|--------------|
| Management Groups | 2 (root + landing zone) | 10+ |
| Subscriptions | 1 | 6+ |
| Resource Groups | 1 per cluster | Multiple per function |
| Setup Complexity | Low | High |
| Governance Flexibility | Limited | Maximum |
| Cost Tracking | Tag-based | Subscription-based |
| Best For | Single cluster, PoC | Multi-cluster, enterprise |

## Limitations

:::warning Simplified Deployment Limitations
- **No subscription-level isolation** — all resources share billing and quota limits
- **Limited policy granularity** — policies apply to the entire subscription
- **Shared resource quotas** — all workloads compete for subscription limits
- **Migration complexity** — moving to Full CAF/WAF later requires restructuring
:::

## Deliverables
