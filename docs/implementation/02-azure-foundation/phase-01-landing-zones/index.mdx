## Overview

Deploy Azure landing zone architecture including management groups, subscriptions, and resource group structure following Azure Cloud Adoption Framework (CAF) best practices.

All examples on this page use **Infinite azurelocal Corp (IIC)**. Replace names with values from your `variables.yml`.

### What This Accomplishes

- **Management group hierarchy** — governance structure for Azure resources
- **Subscription organization** — creates and organizes Azure subscriptions
- **Resource group structure** — organized resource containers for cluster resources

### Prerequisites

:::info Azure Authentication Required
Before running any scripts in this phase, ensure you have an authenticated Azure session.
See [Authentication](../../05-authentication.mdx) for options including Azure PowerShell, Azure CLI, and service principal authentication.
:::

| Run As | Description |
|--------|-------------|
| ⚡ **Elevated Admin** | Owner or User Access Administrator at tenant root scope |

- **Entra ID Permissions** — User Access Administrator or Owner role at tenant root scope
- **Subscription Access** — Access to create subscriptions or EA/MCA enrollment (full deployment)
- **`variables.yml`** — Configured with management group, subscription, and resource group values

## Choose Your Deployment Model

:::warning Decision Required
Select a deployment model before proceeding. The model determines the management group hierarchy, subscription count, and resource group structure.
:::

### Deployment Model Comparison

| Aspect | Full CAF/WAF Deployment | Simplified Deployment |
|--------|-------------------------|----------------------|
| **Management Groups** | Complete hierarchy per CAF Enterprise-Scale (10+) | Root MG + Landing Zone MG (2) |
| **Subscriptions** | Dedicated subscription per function (6+) | Single subscription for all resources |
| **Resource Groups** | Multiple RGs per subscription | Single RG for all cluster resources |
| **Governance** | MG-level policies with inheritance | Subscription-level policies |
| **RBAC** | MG-level role assignments | Resource group-level access control |
| **Cost Tracking** | Subscription-based isolation | Tag-based allocation |
| **Best For** | Enterprise, regulated, multi-cluster | PoC, labs, single cluster, quick-start |
| **Complexity** | Higher | Lower |
| **Setup Time** | ~3 hours | ~1 hour |

## Full CAF/WAF Deployment

**Best for**: Enterprise customers, strict governance requirements, multi-environment production, regulated industries

Deploy complete Azure Landing Zone architecture following Microsoft Cloud Adoption Framework and Well-Architected Framework guidelines.

### Architecture Overview

```
Tenant Root Group
└── cmp-iic-root # Root MG
 ├── cmp-platform-iic # Platform Services
 │ ├── cmp-platform-management-iic
 │ ├── cmp-platform-connectivity-iic
 │ ├── cmp-platform-identity-iic
 │ └── cmp-platform-security-iic
 ├── cmp-landing-zones-iic # Landing Zones
 │ ├── cmp-lz-online-iic
 │ └── cmp-lz-corp-iic
 ├── cmp-sandbox-iic
 └── cmp-decommissioned-iic
```

## Simplified Deployment

**Best for**: PoC/labs, single cluster deployments, cost-conscious customers, rapid deployment timelines

Deploy streamlined landing zone structure with a root management group, a landing zone management group, a single subscription, and a single resource group.

### Architecture Overview

```
Tenant Root Group
└── cmp-iic-root # Root MG
 └── cmp-landing-zones-iic # Landing Zone MG
 └── iic-lz-azurelocal-001 # Subscription
 └── rg-c01-azl-eus-01 # Single resource group
 ├── Azure Local cluster resources
 ├── Arc-enabled servers
 ├── Key Vault
 └── Storage accounts
```

## Deliverables

Regardless of deployment model, the following deliverables are expected at the end of this phase:

- [ ] Management group hierarchy deployed
- [ ] Subscriptions provisioned and associated with management groups
- [ ] Resource groups created following naming conventions from `variables.yml`

## Security Considerations

- **RBAC at scale** — management group-level role assignments (full) or resource group-level (simplified)
- **Policy governance** — centralized policy management and enforcement
- **Resource organization** — logical separation of resources by environment and purpose
- **Cost management** — subscription-level budgeting (full) or tag-based allocation (simplified)
