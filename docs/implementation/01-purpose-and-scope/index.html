Purpose and Scope

DOCUMENT CATEGORY: Runbook
SCOPE: Azure Local deployment guidance
PURPOSE: Define implementation approaches and hardware platform support
MASTER REFERENCE: Microsoft Learn - Azure Local Overview

Status: Active

This runbook provides comprehensive, end-to-end guidance for deploying Azure Local infrastructure. It is designed to support multiple deployment approaches, offering four implementation options for most steps to accommodate different organizational preferences, skill levels, and automation maturity.

Hardware Platform Support​

📋 Current Focus: Dell Hardware

This runbook is currently written for Dell PowerEdge servers with Dell-specific tooling (iDRAC, OpenManage, Dell SBE). Support for Lenovo and DataOn platforms will be added in future releases.

Cross-Platform Compatibility: Approximately 99% of this runbook applies to any Azure Local validated OEM platform. The following sections contain OEM-specific procedures:

SectionOEM-Specific Content
Stage 11: Hardware ProvisioningBIOS/BMC settings and firmware updates
Stage 12: OS InstallationSolution Builder Extension (SBE) package
Stage 18: Monitoring & ObservabilityHardware management extensions (OMIMSWAC)
VariousBMC/iDRAC console procedures

For non-Dell platforms, substitute equivalent OEM tools and procedures.

What This Runbook Covers​

  • CI/CD Infrastructure setup (Stages 00-02)
  • Azure cloud foundation configuration (Stages 03-07)
  • On-premises infrastructure readiness (Stages 08-10)
  • Azure Local cluster deployment (Stages 11-16)
  • Operational foundations and Day 2 configuration (Stages 17-21)
  • Cluster testing and validation
  • Validation and handover
  • Lifecycle operations

What Makes This Runbook Unique​

  • Multi-option format: Each step provides Manual, Direct Script, Orchestrated Script, and Standalone Script options
  • Configuration-driven: Uses variables.yml configuration for portability across deployments
  • Production-ready: Includes validation, troubleshooting, and rollback procedures
  • Framework-aligned: Follows the Documentation Standards and Scripting Framework
