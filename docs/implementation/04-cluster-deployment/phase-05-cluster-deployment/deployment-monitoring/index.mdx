## Overview

Phase 05 cluster deployment runs in two sequential phases ŌĆö validation (~15 minutes) then deployment (~2ŌĆō3 hours). Both phases run silently in the background from Azure's perspective; the portal only updates periodically. These monitoring scripts provide a real-time ANSI dashboard that combines:

- **Azure API polling** ŌĆö live step status from `deploymentSettings/default`
- **Remote log streaming** ŌĆö live tail of `EnvironmentValidatorFull` or `OrchestratorFull` logs from cluster nodes via PSRemoting

Both scripts are auth-method-agnostic ŌĆö they work identically for Active Directory and Local Identity deployments.

## When to Run Each Script

| Phase | Script | When to Start | When It Stops |
|-------|--------|--------------|---------------|
| **Task 9: Validation** | [Monitor Validation](./monitor-validation.mdx) | After clicking **Start validation** in the portal | Auto-exits on validation success or failure |
| **Task 10: Deployment** | [Monitor Deployment](./monitor-deployment.mdx) | After clicking **Create** in the portal | Run until deployment completes (~2ŌĆō3 hours) |

:::info Run on any machine
Both scripts run from **any machine** with network access to the cluster node management IPs and an active Azure authentication context. They do not need to run from a cluster node or the management server used for toolkit scripts.
:::

## Credential Resolution

Both scripts resolve credentials in the same priority order:

```
1. Direct params: -LocalAdminUsername + -LocalAdminPassword
 Ōåō (if not provided)
2. Azure Key Vault: -KeyVaultName with -LocalAdminUsernameSecretName / -LocalAdminPasswordSecretName
 Ōåō (if KV unavailable or not configured)
3. Interactive: Read-Host prompt at startup
```

Azure authentication (for API access) follows the same pattern with SPN credentials:
```
1. -SPNClientId + -SPNClientSecret params
 Ōåō
2. Key Vault: -SPNClientIdSecretName / -SPNClientSecretSecretName
 Ōåō
3. Existing Connect-AzAccount context
```
