Phase 06: Post-Deployment Cluster Configuration

DOCUMENT CATEGORY: Implementation Runbook
SCOPE: Azure Local cluster post-deployment configuration
PURPOSE: Complete post-deployment cluster setup including management tools, security, storage, and logical networks MASTER REFERENCE: Azure Local post-deployment tasks

Status: Active Applies To: Azure Local clusters following cluster deployment (Phase 05) Last Updated: 2026-03-10

Table of Contents​

  • Task 01 – Deploy SDN
  • Task 02 – Cluster Quorum Configuration
  • Task 03 – Security Groups Applied to Nodes
  • Task 04 – SSH Connectivity to Nodes

  • Task 06 – Image Downloads
  • Task 07 – Configure Network Security Groups
  • Task 08 – Logical Network Creation
  • Task 09 – Post-Deployment Verification

Phase Details​

Objective​

Complete post-deployment cluster configuration including management tools, security, storage, and networking infrastructure for workload VM deployment.

Tasks​

  • Deploy Software Defined Networking (SDN) infrastructure including Network Controller, Software Load Balancer, and RAS Gateway (Coming Soon)
  • Configure cluster quorum settings (Cloud Witness, File Share Witness, or Disk Witness)
  • Apply security groups to cluster nodes for RBAC and administrative access control
  • Configure SSH connectivity to cluster nodes for remote administration and troubleshooting
  • Review and configure Storage Spaces Direct pool (capacity, resiliency, health)
  • Create CSV volumes for VM storage with naming conventions, resiliency settings, and provisioning type
  • Create storage paths in Azure mapped to CSV volumes
  • Download and stage VM images (Azure Marketplace or custom images) to storage paths
  • Create logical networks for workload VLANs with VLAN IDs, IP pools, and network properties
  • Verify virtual switch configuration and network intent deployment
  • Run post-deployment verification across all tasks

Outcome​

Cluster fully configured with management tools, security controls, storage infrastructure, and logical networks ready for workload VM deployment.
