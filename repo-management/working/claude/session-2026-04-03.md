## CI Failure — Prettier Formatting

After pushing the new pages and config changes, the `Lint & Format` CI job failed on `prettier --check`.

**Root cause:** Docusaurus/Prettier enforces consistent formatting on all staged files. The 5 new pages were saved without running Prettier first. Additionally, 7 pre-existing files in the repo had accumulated formatting drift and also failed the check.

**Fix:** Ran Prettier write pass across all JS, JSX, JSON, CSS, and MD files:

```bash
npx prettier --write "src/**/*.{js,jsx,json,css,md}"
```

Then committed the formatted output.

**Commit:** `6aa4fd76`

---

## Lessons Learned

### Prettier CI enforcement

- Always run `npx prettier --write` before committing any new or edited files in this repo.
- The CI `prettier --check` step will fail on ANY file with formatting drift — including pre-existing files that happen to be in scope.
- The safe command to format everything relevant: `npx prettier --write "src/**/*.{js,jsx,json,css,md}"`
