This guide provides examples of common MDX (Markdown + JSX) features and components used in TierPoint documentation. Use these patterns to ensure consistency and clarity across all standards documents.

---

## Headings

```mdx
# H1 Heading
## H2 Heading
### H3 Heading
```

---

## Paragraphs and Line Breaks

Separate paragraphs with a blank line. Use `<br />` for a line break within a paragraph.

---

## Bold, Italic, and Inline Code

```mdx
**Bold text**
*Italic text*
`Inline code`
```

## Admonitions (Callout Boxes)

Docusaurus supports special callout/admonition blocks for notes, tips, warnings, etc.

```mdx
:::note
This is a note.
:::

:::tip
This is a tip.
:::

:::info
This is an info box.
:::

:::caution
This is a caution.
:::

:::danger
This is a danger/warning.
:::
```
