## Overview

This stage validates that customer/partner enterprise network infrastructure prerequisites are complete and ready for Azure Local deployment. **No configuration is performed in this stage** - only validation and documentation of readiness.

:::warning Validation Only
This stage verifies that the customer's enterprise network team has completed all prerequisite configuration tasks. If validation fails, work with the customer to resolve issues before proceeding to Phase 03.
:::

:::info Discovery Information Required
This validation stage verifies network infrastructure that was **already documented during customer discovery**. Before beginning Phase 02 validation:

- **Discovery documentation must be complete**: Customer discovery sessions, workshops, and Azure Local training documented
- **Configuration files must be populated**: All network requirements, IP allocations, DNS servers, NTP servers, VLAN IDs, and infrastructure details captured
- **Customer questionnaire must be completed**: Discovery questionnaire filled out by customer with complete network design

**You are validating what was designed during discovery** - if information is missing from configuration files, return to discovery phase before attempting validation.
:::

---

## What This Stage Validates

| Area | Validation |
|------|------------|
| **Physical Infrastructure** | Hardware racked, powered, cabled per design |
| **Customer DNS Services** | Resolution to AD domain controllers |
| **Customer NTP Services** | Time synchronization within 5 minutes |
| **OpenGear Console** | Console server enrolled with Lighthouse portal |

:::note Azure Endpoint Testing
Azure and Dell endpoint connectivity testing occurs in **Phase 03 Step 4** after network infrastructure (switches, firewall rules) is configured.
:::

---

## Stage Steps

| Step | Description | Time | Prerequisites |
|------|-------------|------|---------------|
| 1 | Visual Hardware Inspection | 30 min | Physical access to site |
| 2 | Customer Network Service Verification | 30 min | Management network access |
| 3 | OpenGear Lighthouse Verification | 15 min | Lighthouse portal access |
| 4 | Validation Sign-Off | 30 min | All previous steps complete |

---

## Equipment Required

- Management laptop/workstation with PowerShell 7+ installed
- Network connectivity to customer management VLAN (where nodes will be deployed)
- Access to OpenGear Lighthouse portal
- Azure Local node iDRAC/BMC IP addresses (from configuration files)

:::warning Validation Location
**Validation MUST run from a device on the same management VLAN as the planned Azure Local nodes.** Nodes are not yet deployed, so validation cannot run from cluster nodes.
:::

---

## Prerequisites

Before starting this stage, verify:

- [ ] Customer discovery sessions completed and documented
- [ ] Network architecture documented with IP allocations
- [ ] All hardware racked, powered, and cabled per vendor specifications
- [ ] Customer network team has completed DHCP, DNS, NTP, firewall configuration
- [ ] OpenGear console server racked, powered, and connected to internet
- [ ] Management VLAN accessible from validation workstation
- [ ] Credentials available for validation workstation and Lighthouse portal

---

## Outcome

**Success Criteria:**
- All validation checklist items marked complete
- Customer-provided DNS and NTP services verified operational
- OpenGear appears in Lighthouse portal with active connection
- Customer sign-off obtained

**Result:** Customer network infrastructure prerequisites validated. Site ready to proceed to Phase 03 (Network Infrastructure Configuration) where switches, firewall rules, and Azure endpoint connectivity will be configured and tested.

---

## Validation Checklist

- [ ] All servers physically installed and powered
- [ ] Network cabling matches design specifications
- [ ] DNS resolution working for AD domain controllers
- [ ] NTP synchronization verified (< 5 min skew)
- [ ] OpenGear registered in Lighthouse portal
- [ ] Customer sign-off obtained
